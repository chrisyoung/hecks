# TypeScript Type Generation

Generate `.d.ts` type definitions from your Hecks domain, so frontend clients
get typed API contracts that stay in sync with the backend.

## Automatic generation on build

When you run `hecks build`, a `docs/types.d.ts` file is generated alongside
OpenAPI, JSON Schema, and glossary artifacts:

```ruby
Hecks.build(domain, version: "2026.04.01.1")
# => pizzas_domain/docs/types.d.ts
```

## Standalone export

Use `hecks dump --types` to export just the TypeScript definitions:

```bash
hecks dump --types
# => Dumped types.d.ts
```

## Programmatic usage

```ruby
require "hecks"

domain = Hecks.domain "Pizzas" do
  aggregate "Pizza" do
    attribute :name, String
    attribute :price, Float
    attribute :toppings, list_of("Topping")
    attribute :category, String, enum: %w[classic gourmet]

    value_object "Topping" do
      attribute :name, String
      attribute :amount, Integer
    end

    command "CreatePizza" do
      attribute :name, String
      attribute :price, Float
    end

    lifecycle :status, default: :draft do
      transition "CreatePizza" => "active"
      transition "ArchivePizza" => "archived"
    end

    command "ArchivePizza"
  end

  aggregate "Order" do
    reference_to "Pizza"
    attribute :quantity, Integer

    command "PlaceOrder" do
      reference_to "Pizza"
      attribute :quantity, Integer
    end
  end
end

puts Hecks::HTTP::TypescriptGenerator.new(domain).generate
```

## Example output

```typescript
// Generated by Hecks -- Pizzas domain types
// Do not edit manually

export interface Topping {
  name: string;
  amount: number;
}

export enum PizzaStatus {
  Draft = "draft",
  Active = "active",
  Archived = "archived"
}

export interface Pizza {
  id: string;
  name: string;
  price: number;
  toppings: Topping[];
  category: "classic" | "gourmet";
  createdAt: string;
  updatedAt: string;
}

export type CreatePizza = {
  name: string;
  price: number;
};

export type CreatedPizza = {
  occurredAt: string;
  name: string;
  price: number;
};

export interface Order {
  id: string;
  quantity: number;
  pizza: string; // Reference to Pizza
  createdAt: string;
  updatedAt: string;
}

export type PlaceOrder = {
  quantity: number;
  pizza: string; // Reference to Pizza
};
```

## Type mapping

| Hecks type | TypeScript type            |
|------------|----------------------------|
| String     | `string`                   |
| Integer    | `number`                   |
| Float      | `number`                   |
| Boolean    | `boolean`                  |
| Date       | `string`                   |
| DateTime   | `string`                   |
| JSON       | `Record<string, unknown>`  |
| list_of(T) | `T[]`                      |
| enum       | string union               |
| reference  | `string` (UUID)            |
| lifecycle  | `enum`                     |
